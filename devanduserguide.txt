1. INTRODUCTION

1.1. What is RAG?

RAG (Retrieval-Augmented Generation) is a technique that combines:
┌─────────────────────────────────────────────────────────────────────────────┐
│  RETRIEVAL              +              GENERATION                           │
│  ─────────                             ──────────                           │
│  Search knowledge base                 Generate answer using LLM            │
│  (Qdrant Vector DB)                    (Gemini / Groq)                     │
│                                                                             │
│  Result: Accurate, context-aware answers with sources                      │
└─────────────────────────────────────────────────────────────────────────────┘

1.2. Why RAG for Legal Cases?

┌─────────────────────────────────────────────────────────────────────────────┐
│  LEGAL CHALLENGE                    RAG SOLUTION                            │
│  ────────────────                   ─────────────                           │
│  ✅ Case-specific context           → Vector search for relevant info       │
│  ✅ Need for accurate sources       → Citations and references              │
│  ✅ Legal terminology               → Domain-specific knowledge base        │
│  ✅ Complex queries                 → Hybrid search (vector + keyword)      │
│  ✅ Evolving knowledge              → Update knowledge base anytime          │
└─────────────────────────────────────────────────────────────────────────────┘

│  │  - Extracts keywords and topics                                    │   │
│  │  - Determines confidence scores                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  LEGAL RESEARCHER                                                   │   │
│  │  ─────────────────                                                  │   │
│  │  - Searches Qdrant for relevant legal knowledge                    │   │
│  │  - Returns chunks with relevance scores                            │   │
│  │  - Supports semantic search                                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  CASE CONTEXT PROVIDER                                              │   │
│  │  ─────────────────────                                             │   │
│  │  - Fetches case details from Bharat API                            │   │
│  │  - Provides case context to RAG pipeline                           │   │
│  │  - Formats case data for LLM                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  RAG PIPELINE                                                       │   │
│  │  ─────────────                                                      │   │
│  │  - Orchestrates the entire RAG flow                                │   │
│  │  - Combines legal + case context                                   │   │
│  │  - Calls response generator                                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  RESPONSE GENERATOR                                                 │   │
│  │  ───────────────────                                                │   │
│  │  - Generates final answer using LLM                                │   │
│  │  - Formats response with sources                                   │   │
│  │  - Supports multi-language                                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  DOCUMENT PROCESSOR                                                 │   │
│  │  ───────────────────                                                │   │
│  │  - Processes PDF uploads                                           │   │

│     │  class QdrantService:                                           │   │
│     │      def search(self, query: str) -> List[Dict]:               │   │
│     │          # Implementation                                      │   │
│     └─────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  3. Pipeline Pattern                                                       │
│     ┌─────────────────────────────────────────────────────────────────┐   │
│     │  class RAGPipeline:                                             │   │
│     │      def process(self, query: str) -> Dict:                    │   │
│     │          # Step 1: Classify                                    │   │
│     │          # Step 2: Search                                      │   │
│     │          # Step 3: Generate                                    │   │
│     └─────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

6.2. FAQ

┌─────────────────────────────────────────────────────────────────────────────┐
│  Q: How does RAG work?                                                    │
│  A: RAG combines document retrieval with LLM generation. It searches       │
│     the knowledge base (Qdrant) for relevant chunks and then sends         │
│     them to the LLM to generate a context-aware answer.                   │
│                                                                             │
│  Q: What is Qdrant?                                                       │
│  A: Qdrant is a vector database. It stores embeddings of text chunks       │
│     and enables fast semantic similarity search.                          │
│                                                                             │
│  Q: Can I use any LLM?                                                    │
│  A: Yes. Currently supported: Gemini (primary) and Groq (fallback).        │
│     You can add more by extending LLMGateway.                             │
│                                                                             │
│  Q: How much storage does the knowledge base need?                         │
│  A: Depends on document size. The free tier of Qdrant gives 4 GB,          │
│     which is enough for ~1 million vectors.                               │
│                                                                             │
│  Q: How do I backup my knowledge base?                                    │
│  A: Use Qdrant snapshots or export vectors to JSON.                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
